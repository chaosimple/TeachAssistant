# 课堂点名功能实现计划

## 需求概述
在"课堂互动"页面增加"课堂点名"功能，按导入顺序依次点所有同学的名字，只为未到的学生记录缺席（0分），到场的学生不做记录。

## 现有架构分析
- **课堂互动页面**：已有随机点名功能（`js/attendance.js`）
- **数据结构**：
  - 学生数据：`localStorage.classroom_students` (数组，含id、name、rollCount)
  - 互动记录：`localStorage.classroom_interaction` (数组，含date、time、studentId、studentName、score、week)

## 实现方案

### 1. HTML 修改（index.html）
在"课堂互动"标签页增加：
- "课堂点名"按钮（位于随机点名按钮旁边）
- 课堂点名面板（id: classrollPanel），包含：
  - 标题和进度显示
  - 当前学生姓名和学号显示
  - "到场"和"缺席"按钮
  - 上一位/下一位导航按钮
  - 结束点名/取消按钮
  - 缺席学生列表显示

### 2. JavaScript 修改（js/attendance.js）
新增以下功能：

#### 2.1 课堂点名状态管理
```javascript
classRollCall: {
    isActive: false,           // 是否正在进行课堂点名
    students: [],              // 学生列表（按导入顺序）
    currentIndex: 0,          // 当前点到第几个
    absentList: []            // 缺席学生列表 [{studentId, studentName}]
}
```

#### 2.2 新增方法
- `startClassRollCall()` - 开始课堂点名
- `renderCurrentStudent()` - 渲染当前学生信息
- `renderAbsentList()` - 渲染缺席列表
- `markCurrentPresent()` - 标记当前学生到场（不记录）
- `markCurrentAbsent()` - 标记当前学生缺席，记录0分，立即保存到历史
- `nextStudent()` - 下一位学生
- `prevStudent()` - 上一位学生
- `finishRollCall()` - 结束点名，保存所有缺席记录
- `cancelRollCall()` - 取消点名
- `closeClassRollCall()` - 关闭点名面板

#### 2.3 数据存储
缺席记录保存到 `classroom_interaction`：
```javascript
{
    date: '2026-04-06',
    time: '09:30:00',
    studentId: '001',
    studentName: '张三',
    score: 0,  // 缺席为0分
    week: 5,
    type: 'rollcall' // 标记来源为课堂点名
}
```

### 3. CSS 修改（css/style.css）
新增课堂点名面板样式：
- `.classroll-panel` - 面板主体，深灰色背景(#374151)
- `.classroll-header` - 标题栏
- `.classroll-current` - 当前学生显示区域
- `.classroll-buttons` - 到场/缺席按钮
- `.classroll-nav` - 导航按钮
- `.classroll-actions` - 结束/取消按钮
- `.classroll-absent-list` - 缺席列表显示

### 4. 关键文件修改
- `index.html`：增加点名相关UI元素
- `js/attendance.js`：增加课堂点名逻辑
- `css/style.css`：增加点名面板样式

## 验证方式
1. 点击"课堂点名"按钮
2. 界面显示所有学生列表（按导入顺序）
3. 依次对每个学生点击"到场"或"缺席"
4. 点击"结束点名"保存记录
5. 在"历史记录"页面查看缺席记录（0分）
6. 确认数据正确保存到localStorage
